Configuração Firebase + Vercel

Guia completo para integrar o Firebase ao seu projeto Next.js e fazer deploy no Vercel

Configuração do Firebase

Criar Projeto no Firebase

Passos para configurar um novo projeto no Firebase Console

  • Acesse console.firebase.google.com
  • Clique em "Criar projeto" ou "Adicionar projeto"
  • Defina um nome para seu projeto
  • Configure o Google Analytics (opcional)

Configurar Firestore

Como configurar o banco de dados Firestore

  • No menu lateral, clique em "Firestore Database"
  • Clique em "Criar banco de dados"
  • Escolha as regras de segurança (iniciar no modo teste)
  • Selecione a localização do servidor

Configurar Autenticação

Habilitar métodos de autenticação no Firebase

  • No menu lateral, clique em "Authentication"
  • Vá para a aba "Sign-in method"
  • Habilite Provedor de e-mail/senha
  • Configure Google e GitHub como provedores OAuth

Configuração do Código

Instalação de Dependências

Terminal 
# Instalar dependências do Firebase
npm install firebase
# ou
yarn add firebase

# Instalar dependências para o Next.js (se necessário)
npm install next@latest react@latest react-dom@latest
npm install --save-dev @types/node

Configuração do Firebase

lib/firebase.js 
// Importar functions do Firebase
import { initializeApp } from "firebase/app";
import { getAuth } from "firebase/auth";
import { getFirestore } from "firebase/firestore";

// Configuração do Firebase
const firebaseConfig = {
  apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
  authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
  projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
  storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
  messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
  appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID
};

// Inicializar Firebase
const app = initializeApp(firebaseConfig);

// Inicializar Authentication e Firestore
const auth = getAuth(app);
const db = getFirestore(app);

export { auth, db };
export default app;

Passos para Configuração

1

Configurar Variáveis de Ambiente

Crie um arquivo .env.local na raiz do seu projeto Next.js e adicione as variáveis de ambiente do Firebase:

NEXT_PUBLIC_FIREBASE_API_KEY=sua_api_key_aqui
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=seu_projeto.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=seu_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=seu_projeto.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=seu_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=sua_app_id
2

Configurar Regras do Firestore

No Firebase Console, vá para Firestore Database > Regras e configure as regras de segurança:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Permite leitura/gravação apenas para usuários autenticados
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
3

Configurar Deploy no Vercel

Conecte seu repositório GitHub ao Vercel e configure as variáveis de ambiente:

  • Acesse vercel.com e faça login com sua conta GitHub
  • Clique em "Import Project" e selecione seu repositório
  • Configure as variáveis de ambiente na seção "Environment Variables"
  • Deixe as configurações padrão para um projeto Next.js
  • Clique em "Deploy"

Configuração do Vercel

Conectar Repositório

Como conectar seu repositório GitHub ao Vercel

  • Faça login no Vercel com sua conta GitHub
  • Clique em "New Project"
  • Importe o repositório do seu projeto
  • Configure o framework como Next.js

Variáveis de Ambiente

Configurar as variáveis de ambiente no Vercel

  • No dashboard do projeto, vá em Settings
  • Selecione a aba "Environment Variables"
  • Adicione todas as variáveis do Firebase
  • Certifique-se de adicionar para ambiente de produção

Deploy Automático

Configurar deploy automático com GitHub

  • O Vercel faz deploy automático a cada push no GitHub
  • Preview deployments para cada pull request
  • Rollback fácil para versões anteriores
  • Domínio personalizável
